# Plan Docente - Ilustración Aplicada: Productos Digitales Web-App

## Contexto y perfil del estudiante

Este módulo está diseñado para estudiantes de **Ilustración Aplicada** que desean expandir sus habilidades creativas al entorno digital, centrándose en el desarrollo de productos web y aplicaciones interactivas. El curso asume que los estudiantes tienen una sólida base en principios visuales y de diseño, pero poca o ninguna experiencia en programación.

### Perfil del estudiante ideal

- **Formación artística**: Estudiantes con background en ilustración, diseño gráfico o artes visuales
- **Interés digital**: Curiosidad por cómo sus habilidades creativas pueden aplicarse a medios interactivos
- **Nivel técnico**: Principiantes en programación, pero con disposición para aprender
- **Objetivos profesionales**: Interés en trabajar como diseñadores UX/UI, desarrolladores front-end creativos, o artistas digitales

## Objetivos del módulo

Al finalizar este módulo, los estudiantes serán capaces de:

### Fundamentos técnicos

- **Comprender los principios básicos** del desarrollo web moderno (HTML, CSS, JavaScript)
- **Dominar herramientas de desarrollo** y flujos de trabajo profesionales
- **Implementar diseños responsivos** que funcionen en múltiples dispositivos
- **Aplicar principios de accesibilidad** y usabilidad web

### Aplicaciones creativas

- **Transformar ilustraciones estáticas** en experiencias web interactivas
- **Crear animaciones y transiciones** que mejoren la narrativa visual
- **Desarrollar interfaces atractivas** y funcionales para proyectos digitales
- **Integrar elementos multimedia** en proyectos web complejos

### Desarrollo profesional

- **Construir un portafolio digital** que showcase sus habilidades técnicas y creativas
- **Colaborar en equipos de desarrollo** utilizando metodologías ágiles
- **Comprender el ciclo completo** del desarrollo de productos digitales
- **Evaluar críticamente** las implicaciones éticas y sociales del diseño digital

## Metodología ATELIER

Este módulo sigue la metodología **ATELIER**, que se caracteriza por:

### Aprendizaje experiencial

- **Proyecto central**: Cada estudiante desarrolla un producto web completo a lo largo del módulo
- **Iteración continua**: Feedback constante y mejora progresiva
- **Reflexión crítica**: Análisis del proceso y resultados en cada etapa

### Estructura de sesiones

#### Sesión 1: Fundamentos y Conceptualización

**Duración**: 4 horas

**Contenido**:
- Introducción al desarrollo web moderno
- Conceptos básicos de HTML y CSS
- Configuración del entorno de desarrollo
- Inicio del proyecto personal

**Actividades**:
- Configuración de herramientas de desarrollo
- Creación de la primera página web
- Definición del alcance del proyecto personal
- Peer review de ideas iniciales

#### Sesión 2: Diseño Responsivo y Frameworks

**Duración**: 4 horas

**Contenido**:
- Principios de diseño responsivo
- Introducción a frameworks CSS (Bootstrap y Tailwind)
- Optimización para dispositivos móviles
- Branding personal y presentación profesional

**Actividades**:
- Implementación de layouts responsivos
- Experimentación con frameworks CSS
- Desarrollo de la identidad visual del proyecto
- Presentación de avances

#### Sesión 3: Interactividad y UX/UI

**Duración**: 4 horas

**Contenido**:
- Fundamentos de JavaScript
- Principios de UX/UI design
- Animaciones y micro-interacciones
- Testing y optimización

**Actividades**:
- Implementación de interactividad básica
- Diseño de flujos de usuario
- Creación de animaciones significativas
- Testing con usuarios reales

## Evaluación

La evaluación sigue un enfoque **formativo y continuo**, con énfasis en:

### Criterios de evaluación

#### Componente técnico (40%)

- **Calidad del código**: Estructura, semántica y buenas prácticas
- **Funcionalidad**: Completitud y correcto funcionamiento de features
- **Responsividad**: Adaptación a diferentes dispositivos y pantallas
- **Accesibilidad**: Cumplimiento de estándares WCAG

#### Componente creativo (40%)

- **Diseño visual**: Coherencia estética y atractivo visual
- **Experiencia de usuario**: Flujos intuitivos y usabilidad
- **Innovación**: Originalidad en la aplicación de conceptos
- **Narrativa digital**: Integración efectiva de elementos visuales

#### Componente procesional (20%)

- **Documentación**: Calidad y completitud de la documentación del proceso
- **Reflexión crítica**: Análisis profundo de decisiones y aprendizajes
- **Colaboración**: Participación activa en actividades grupales
- **Presentación**: Claridad y efectividad en la comunicación del proyecto

### Momentos de evaluación

1. **Checkpoint 1** (Sesión 1): Evaluación de conceptualización y setup técnico
2. **Checkpoint 2** (Sesión 2): Evaluación de diseño responsivo y branding
3. **Checkpoint 3** (Sesión 3): Evaluación de interactividad y UX final
4. **Entrega final**: Presentación completa del proyecto con reflexión crítica

## Entregables

### Proyecto principal

- **Producto web funcional**: Aplicación o sitio web completo y desplegado
- **Documentación técnica**: README con especificaciones y guía de uso
- **Portafolio de proceso**: Evolución del proyecto con justificaciones de decisiones
- **Video demo**: Presentación del producto funcionando

### Entregables de proceso

- **Bitácora de desarrollo**: Registro diario de avances y desafíos
- **Peer reviews**: Feedback constructivo a proyectos de compañeros
- **Reflexiones semanales**: Análisis crítico del aprendizaje y proceso

## Recursos y herramientas

### Tecnologías principales

- **HTML5**: Estructura semántica y contenido
- **CSS3**: Estilos, layouts y animaciones
- **JavaScript ES6+**: Interactividad y lógica de aplicación
- **Git/GitHub**: Control de versiones y colaboración

### Frameworks y librerías

- **Bootstrap**: Framework CSS para desarrollo rápido
- **Tailwind CSS**: Utility-first CSS framework
- **GSAP**: Librería para animaciones avanzadas
- **Figma/Sketch**: Herramientas de diseño UI/UX

### Plataformas de despliegue

- **GitHub Pages**: Hosting gratuito para sitios estáticos
- **Netlify**: Despliegue continuo y hosting moderno
- **Vercel**: Plataforma optimizada para aplicaciones web

## Referencias y lecturas

### Fundamentos técnicos

- **MDN Web Docs**: Documentación completa de tecnologías web
- **CSS-Tricks**: Guías y tutoriales de CSS y diseño web
- **JavaScript.info**: Recursos comprehensivos de JavaScript
- **A List Apart**: Artículos sobre diseño y desarrollo web

### Diseño y UX

- **Don't Make Me Think** (Steve Krug): Fundamentos de usabilidad web
- **Refactoring UI** (Adam Wathan): Principios de diseño de interfaces
- **Microinteractions** (Dan Saffer): Diseño de detalles interactivos
- **Design Systems** (Alla Kholmatova): Creación de sistemas de diseño consistentes

### Desarrollo profesional

- **The Pragmatic Programmer** (Andrew Hunt): Buenas prácticas de desarrollo
- **Clean Code** (Robert C. Martin): Principios de código limpio
- **Designing Data-Intensive Applications** (Martin Kleppmann): Arquitectura de sistemas

## Nota crítica

Este módulo busca trascender la simple adquisición de habilidades técnicas para fomentar una **reflexión crítica** sobre el rol del diseño y la tecnología en la sociedad contemporánea. Los estudiantes serán incentivados a:

- **Cuestionar paradigmas establecidos** en el diseño digital
- **Considerar implicaciones éticas** de sus decisiones de diseño
- **Desarrollar una voz auténtica** en su trabajo creativo
- **Contribuir a un ecosistema digital** más inclusivo y accesible

El enfoque ATELIER no solo busca formar técnicos competentes, sino **creadores conscientes** que puedan utilizar sus habilidades para generar impacto positivo en el mundo digital.
