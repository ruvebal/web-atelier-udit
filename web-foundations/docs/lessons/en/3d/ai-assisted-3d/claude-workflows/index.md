# Claude Workflows for 3D Web — CLAUDE.md, Subagents & Skills

> *"Teach the AI your architecture once, with care, and it builds within your walls. Teach it only with chat, and it builds wind — pretty, and impossible to live in."*

Claude Code (and similar long-horizon agents) become reliable when **project memory** is file-backed: `CLAUDE.md`, skills, and repeatable subagent prompts. For 3D web work, the cost of ambiguity is high — wrong loader, wrong color space, wrong frame loop — so invest in **boring** documentation the agent can cite.

---

## `CLAUDE.md` for 3D projects (template)

**Template** — Save as `CLAUDE.md` at the repository root (or monorepo package root). **Customize:** product name, React / three.js / R3F versions, real directory paths, npm script names, draw-call and texture budgets, testing commands, and legal / asset paths. Remove sections that do not apply.

```markdown
# CLAUDE.md — Acme 3D Web (R3F + Vite)

## Mission
Production experiential site: React 19, react-router v7, @react-three/fiber, drei, three.js, shadcn/ui.
Ship fast first paint, stable 60 FPS on M-series reference laptop for hero scene.

## Hard rules
- JavaScript only (no TypeScript). Use JSDoc typedefs for complex shapes when helpful.
- No React state updates inside `useFrame` unless throttled/event-driven.
- All new textures for three.js: KTX2 when using compressed runtime path; document exceptions.
- Models: glTF `.glb` only in `public/assets/3d/`. CI runs gltf-transform inspect.
- Always implement `prefers-reduced-motion` path at route level.
- UI components: shadcn/ui in src/components/ui/ — never import three.js. Drei Html overlays in src/components/canvas/overlays/.
- Follow DESIGN.md for palette, typography, spacing, and motion tokens.

## Directory map
- `src/components/ui/` — DOM components; never import three.
- `src/components/canvas/` — Canvas, scenes, R3F providers.
- `src/components/canvas/overlays/` — Drei Html bridges to DOM components.
- `src/lib/three/` — math, materials, loaders, non-React utilities.
- `scripts/` — asset pipelines (Node).

## Commands
- `npm run dev` — Vite dev
- `npm run build` — production build
- `npm run lint` — eslint
- `npm run assets:inspect` — gltf-transform stats for CI parity

## Design system
- DESIGN.md at project root defines palette, type scale, spacing, motion.
- Generated by Stitch MCP or authored manually.
- UI UX Pro Max skill informs aesthetic decisions when active.

## Performance budgets (do not exceed without explicit approval)
- Desktop hero: ≤ 200 draw calls, ≤ 2 shadow-casting lights
- Mobile scene: ≤ 80 draw calls; prefer baked lighting
- Textures: 2K max per map unless art sign-off

## Testing expectations
- After WebGL changes, summarize expected impact on draw calls and transparency overdraw.
- If shaders change, note WebGL2 requirement and precision choices.

## Security / licensing
- Do not add external asset CDN URLs without review.
- No copyrighted model packs without license file in `legal/third-party/`.

## When stuck
- Read `docs/3d/PERF_BUDGETS.md` and `docs/3d/ASSET_PIPELINE.md` before proposing new deps.
```

## Designing subagent prompts

Subagents excel when the parent agent delegates **bounded** tasks with **clear inputs and outputs**. The **Template** blocks below are starters—adjust stack names, folder rules, and output format to your agent host.

### Subagent: asset processing pipeline

**Template** — Replace generic paths and platform list with your project’s inputs.

```text
You are the asset-pipeline subagent for a three.js R3F project.

Inputs you may receive: path to `.glb`, target platforms (iOS Safari, desktop Chrome), texture budget.

Your job:
1. Propose a gltf-transform command sequence (meshopt/draco decisions explained).
2. Propose KTX2 encoding settings (UASTC vs ETC1S) per texture type (albedo vs normal).
3. Output a final folder layout and npm script additions.
4. Never suggest committing source PSDs to public/.

Output format:
- ## Plan
- ## Commands (shell blocks)
- ## Risks (decode cost, normal map quality, etc.)
```

### Subagent: shader generation

**Template** — Align uniform prefixes and `discard` policy with your `CLAUDE.md` / shader rules.

```text
You are the shader subagent.

Stack: three.js WebGL2, ShaderMaterial with inline GLSL strings in JS.

Rules:
- Prefix uniforms with u*, varyings with v*.
- No discard() unless user explicitly allows transparency cutouts.
- Avoid dynamic loops. Unroll small fixed loops only with justification.
- Include top-of-shader comment: purpose, inputs, spaces, performance notes.

Deliver:
- vertex + fragment as JS template literals ready to paste
- uniform setup snippet for three.js
```

### Subagent: performance audit

**Template** — Paste real counts and scene description into the parent prompt when invoking the subagent.

```text
You are the performance audit subagent.

You will receive: scene description, renderer settings, rough counts (tris, draw calls, lights), optional screenshot description.

Deliver prioritized findings:
- P0 critical (likely jank / mobile failure)
- P1 important
- P2 polish

Each item: symptom → likely cause → fix → how to verify (DevTools / renderer.info).
No generic advice — tie to provided numbers.
```

### Subagent: scene composition

**Template** — Attach the creative brief and brand constraints in the same message.

```text
You are the composition subagent (art direction + three.js staging).

Given a creative brief, propose:
- camera focal language (wide vs long), eye height, dolly vs crane
- lighting triad (key/fill/rim) and shadow strategy
- color grading approach (post stack choices)
- motion choreography on scroll/hover with reduced-motion alternative

Do not write code until user asks; default is written shot list + rationale.
```

### Subagent: UI overlay design

**Template** — Attach DESIGN.md and the target scene description.

```text
You are the UI overlay subagent for an R3F + shadcn/ui project.

Given: scene description, DESIGN.md tokens, overlay requirements.

Deliver:
1. Component list (shadcn/ui primitives: Card, Sheet, Dialog, etc.)
2. Whether each component is DOM overlay or Drei Html (with position/distanceFactor)
3. z-index assignments following DESIGN.md scale
4. Accessibility notes (focus management, reduced-motion alternative)
5. Performance notes (Html instance count, occlude strategy)

Use Stitch MCP to generate the visual design first if connected.
Use DESIGN.md tokens for all color/type/spacing — no arbitrary values.
```

## Claude Code / Codex skills for 3D

**Skills** package repeatable procedures the agent loads on demand. Good 3D skills are **narrow**:

### Skill idea: `ktx2-bake`

Contents should include:

- When to use UASTC vs ETC1S
- Example `toktx` / `basisu` commands (project-specific flags)
- How textures are registered in three.js (`KTX2Loader` + transcoder path)
- A verification checklist (mip chain, color space, normal map swizzle)

### Skill idea: `r3f-scene-starter`

- Minimal `Canvas` snippet with `dpr` policy
- `PerformanceMonitor` downgrade hook template
- File placement under `src/components/canvas/scenes/`

Author skills in your skills directory following the tool’s schema (name, description, when to use, steps). Link to `CLAUDE.md` for values that change rarely.

### Skill skeleton (markdown-friendly)

Use this shape when your tooling expects a prose skill file:

**Template** — Rename the skill, paths, and npm script to match your repository; match frontmatter keys to Claude Code, Codex, or your internal host.

```markdown
---
name: ktx2-bake-acme
description: Encode textures to KTX2 for Acme’s three.js pipeline (Basis/UASTC rules)
---

# When to use
- New PBR surface textures before merge to `public/assets/3d/`
- When normal maps look crunchy after compression

# Inputs
- Source PNG/WebP paths
- Intended material slot (albedo, normal, ORM, emissive)

# Steps
1. Validate dimensions (power-of-two, max 2K unless approved)
2. Encode albedo as sRGB; data maps as linear (document in sidecar JSON)
3. Run project script `npm run ktx2:encode -- --input ...`
4. Load in three with configured transcoder path; verify in staging build

# Done when
- Visual diff approved on M-series + one Android device
- File size logged in `docs/perf/textures.csv`
```

Adapt frontmatter keys to match **Claude Code**, Codex, or internal skill hosts.

## Parent agent ↔ subagent handoff pattern

1. Parent reads `CLAUDE.md` and recent diffs
2. Parent spawns subagent with **input bundle**: file paths, numbers, constraints
3. Subagent returns **structured** output (headings, tables, command blocks)
4. Parent **verifies**: runs build, applies only approved hunks, updates docs if conventions shifted

Avoid nesting subagents deeply — context fragmentation compounds in 3D repos.

## Memory management for project conventions

- **Canonical:** `CLAUDE.md`, `docs/3d/*`, Cursor/Windsurf rules
- **Ephemeral:** chat — re-read files each session; do not trust “remembered” API names
- **Team ritual:** when a convention changes, update `CLAUDE.md` in the **same PR** as the code

### Convention change template (paste into PR description)

**Template** — Fill brackets when you change budgets or defaults.

```text
## Convention update
- Changed: [tone mapping default / texture max size / …]
- Reason: [measured issue + link to trace or ticket]
- Docs: CLAUDE.md § [x], docs/3d/PERF_BUDGETS.md updated: y/n
- Rollout: [feature flag / immediate]
```

## Example workflows

### “Generate a dissolve shader”

1. Paste material factory location and naming conventions
2. Specify mesh uses transparency cutout vs opacity
3. Ask for GLSL + uniform wiring + one debug GUI knob (`uProgress`)

### “Optimize this glTF”

1. Paste `gltf-transform inspect` output
2. State target: mobile Safari + desktop Chrome
3. Ask for ordered steps with expected byte and decode impact

### “Review this scene for performance”

1. Paste `renderer.info` snapshot and light list
2. Note post-processing stack
3. Demand **measurement** suggestions, not vibes

**CodeSandbox-ready** — Dev-only `console.table`; pass your three.js `WebGLRenderer`.

```js
// Snippet to log renderer info during dev reviews
export function logRendererInfo(renderer) {
  const ri = renderer.info;
  console.table({
    calls: ri.render.calls,
    triangles: ri.render.triangles,
    points: ri.render.points,
    lines: ri.render.lines,
    geometries: ri.memory.geometries,
    textures: ri.memory.textures,
  });
}
```

## Telemetry and critique loops

Wire **client-side** perf beacons sparingly: long tasks, WebGL context loss, frame time samples bucketed — avoid PII. Use the data to answer: “Did our last shader change regress low-end Android?” Feed anonymized summaries back into `docs/3d/PERF_BUDGETS.md` when thresholds move.

**CodeSandbox-ready** — Pass the WebGL canvas and your analytics callback.

```js
// Ultra-minimal context-loss hook — extend with your analytics adapter
export function watchWebglContextLoss(canvas, onLost) {
  canvas.addEventListener('webglcontextlost', (e) => {
    e.preventDefault();
    onLost({ reason: 'webglcontextlost' });
  });
}
```

Pair with a user-visible message and a **non-WebGL** fallback path when possible.

## Troubleshooting Claude Code on shaders

| Symptom | Likely cause | Fix |
|---------|----------------|-----|
| Black screen, no errors | Wrong color space / tone mapping | Verify `outputColorSpace`, renderer settings |
| Z-fighting | Near/far + scale | Adjust camera clip planes; model scale |
| Shimmering specular | Normal map too strong in mip chain | Regenerate mips; check tangent space |
| Works on desktop only | `highp` required | Add precision directive or simplify |

Attach **screenshots** and **uniform values** when asking for help — words alone mislead.

## Review checklist for subagent output (before apply)

1. **Factual** — API names match installed packages; no invented paths
2. **Scoped** — diff touches only the files you listed
3. **Disposed** — textures/geometries have a lifecycle story
4. **Tested** — answer includes a verification step you can run in under 5 minutes
5. **Documented** — if it introduces a new convention, `CLAUDE.md` or `docs/3d/` was updated (or explicitly deferred with ticket ID)

Reject the bundle if 1 or 2 fails; iterate on 3–5 with smaller prompts.

## Versioning subagent prompts in-repo

Store canonical prompts under `docs/agents/prompts/` as markdown. Use git blame to learn **why** a prompt changed (“Safari tangent regression”, “new meshopt path”). Chat history is not a changelog.

Suggested filenames:

- `subagent-asset-pipeline.md`
- `subagent-shader.md`
- `subagent-perf-audit.md`
- `subagent-composition.md`
- `subagent-ui-overlay.md`

Point `CLAUDE.md` to this folder in one line so new teammates discover it.

## AI notes

Subagents reduce context pollution but can **drift** if not anchored to repo files. Always attach manifests, budgets, and the exact three.js version from `package.json` when debugging precision or lighting.
